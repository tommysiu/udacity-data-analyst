Section 1. Statistical Test

1.1 Which statistical test did you use to analyze the NYC subway data? Did you use a one-tail or a two-tail P value? What is the null hypothesis? What is your p-critical value?

Two-tailed Mann Whitney U test has been used to analyze the data. The null hypothesis is that there is no statistical difference between the ridership when it rains and when it does not. A p-critical value Î± = 0.05 is used.

1.2 Why is this statistical test applicable to the dataset? In particular, consider the assumptions that the test is making about the distribution of ridership in the two samples.

The two samples do not seem to have normal distribution by looking at their histograms. Thus I choose not to use t-test to test the two samples. Instead, the Mann Whitney U Test should be more appropriate because it is applicable to unknown distribution and the sizes of both raining and non-raining samples are sufficient to run the test.

1.3 What results did you get from this statistical test? These should include the following numerical values: p-values, as well as the means for each of the two samples under test.

The mean entries per hour for raining days is 1105.45.
The mean entries per hour for non-raining days is 1090.28.
The p-value (two-tail) is 0.03862 after doubling the result from Scipy (scipy v0.15.1 returns 0.019 for one-tail p-value, which corrects the test for tied ranks). Please note that the result is different from the Udacity simulator in Problem Set 3 (scipy returns 0.025) which seems to ignore tied ranks.

1.4 What is the significance and interpretation of these results?

As the p-value (0.03862) is smaller than the p-critical value 0.05, we would reject the null hypothesis. The interpretation is that the distribution of the number of entries is statistically different between rainy and non-rainy days.

Valid R2 value should be from 0 to 1, where 0 means the model does not fit the data and 1 means the model fits the observed data perfectly. 5.1 Please discuss potential shortcomings of the methods of your analysis, including:

  1. Dataset,
  2. Analysis, such as the linear regression model or statistical test.


The dataset only covers 31 days (1 May 2011 - 31 May 2011), which seems a bit short if we want to analyze the effect of raining on ridership. Though there are more than 40000 data points (in the enhanced dataset), the rain variable is set in day level so there is a chance that it actually does not record the real raining situation in hour level. Moreover, my analysis does not try to identify any outliners. This may introduce incorrect results for the statistical test and linear regression. If outliners are removed from our dataset, the linear regression model may produce a more accurate prediction for those non-outliner data points.
As we can see from previous section, the residual does not have a normal distribution, which is a significant issue because the linear regression model may not give accurate predictions. One potential problem of our model is that the correlation of the feature variables. That may cause the multicollinearity issue as it is possible that those variables are linearly related to each other. One impact of the multicollinearity problem is that it tends to produce large standard error of those affected coefficients. Another potential explanation of the non-normal distribution is that the ridership is probably a non-linear combination of a set of independent variables, so a linear regression model is not appropriate.
